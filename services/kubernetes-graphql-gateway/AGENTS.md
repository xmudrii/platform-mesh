# kubernetes-graphql-gateway

Go service that exposes Kubernetes resources as a GraphQL API. Dual-component architecture: Listener (schema extraction) + Gateway (GraphQL serving). Built on `controller-runtime`, `graphql-go`, and `sigs.k8s.io/multicluster-runtime`.

Read the org-wide [AGENTS.md](https://github.com/platform-mesh/.github/blob/main/AGENTS.md) for general conventions. Read [CONTRIBUTING.md](CONTRIBUTING.md) for contribution workflow.

## Commands

- `task validate` ‚Äî generate + lint + test (CI gate)
- `task generate` ‚Äî regenerate CRD manifests and deepcopy from API types
- `task lint` ‚Äî golangci-lint v2 (includes formatting)
- `task test` ‚Äî unit tests with coverage (uses envtest)
- `task proto` ‚Äî regenerate protobuf files
- `task docker` ‚Äî build container image
- `task docker:kind` ‚Äî build and load into kind cluster
- `task gateway` ‚Äî start gateway dev server (port 8080)
- `task listener` ‚Äî start listener dev server

## Architecture

Single binary (`main.go`) with cobra subcommands. Listener and gateway communicate via gRPC (recommended) or file-based schema transport:

- **Listener**: watches Kubernetes clusters, extracts OpenAPI v3 specs, converts to GraphQL schemas, streams to gateway. Supports three provider modes: `single` (local cluster), `kcp` (kcp workspaces via APIExport), `multi` (kcp + standard clusters combined).
- **Gateway**: receives schemas, builds per-cluster GraphQL endpoints at `/api/clusters/{name}/graphql`. Serves queries, mutations, and subscriptions (SSE). Handles auth via Kubernetes TokenReview.

`ClusterAccess` CRD manages remote cluster connections with four auth methods: ServiceAccount, Bearer Token, Kubeconfig, Client Certificate (mTLS).

## Code Conventions

License header on all `.go` files ‚Äî Apache-2.0, "The Platform Mesh Authors". Enforced by goheader linter.

Import order (enforced by gci):
1. stdlib
2. third-party
3. `k8s.io/`, `sigs.k8s.io/`
4. `sigs.k8s.io/multicluster-runtime/`
5. `github.com/kcp-dev/`
6. `github.com/kcp-dev/multicluster-provider/`

Tests: testify (require/assert), testify/suite for e2e. Integration tests use controller-runtime envtest with CRDs loaded from `config/crd/`. Mocks in `listener/pkg/apischema/mocks/` are auto-generated.

All configuration is via CLI flags ‚Äî no environment variables.

Run `task generate` after any change to `apis/v1alpha1/` types.
Run `task proto` after any change to `apis/proto/`.

## Do Not

- Edit `zz_generated.deepcopy.go` ‚Äî generated by controller-gen
- Edit `config/crd/*.yaml` directly ‚Äî generated by `task generate`
- Edit `config/kcp/*.yaml` directly ‚Äî generated by `task generate`
- Edit files in `sdk/` ‚Äî generated from protobuf by `task proto`
- Edit files in `listener/pkg/apischema/mocks/` ‚Äî auto-generated mocks
- Skip the license header ‚Äî CI enforces it
- Add tool-specific instruction files (`CLAUDE.md`, `.cursorrules`, etc.) ‚Äî use this file
- Commit without running `task generate` after API type changes
